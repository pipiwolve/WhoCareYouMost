# MedicalAgent 前端联调协议 v1 — 含侧边栏状态机映射

> 本文档在原 `后端接口说明.md` 基础上，补充了**前端侧边栏 Report 进度条**的状态机设计。
> 供 codex 或其他 AI 生成代码时参考。

## 1. 后端接口清单

| 接口 | 方法 | 路径 | Accept |
|---|---|---|---|
| 初始化用户资料 | POST | `/api/v1/users/profile` | `application/json` |
| 聊天（同步JSON） | POST | `/api/v1/chat/completions` | `application/json` |
| 聊天（SSE流式） | POST | `/api/v1/chat/completions` | `text/event-stream` |
| 查询报告 | GET | `/api/v1/reports/{sessionId}` | `application/json` |
| 下载报告 PDF | GET | `/api/v1/reports/{sessionId}/pdf` | `application/pdf` |

### 1.1 代理配置

前端统一请求 `/api/v1/...` 相对路径，由 Next.js rewrites 代理到真实后端。

- 环境变量：`API_PROXY_TARGET`（服务端，不暴露给浏览器）
- 默认值：`http://localhost:8123`

### 1.2 关键约束

- `attachments` 当前必须为空或不传
- SSE 不是模型真流式，后端先拿完整回复，再按中文标点+12字切分推 chunk
- `reportAvailable=true` 不代表 report 字段有值，需单独调用 getReport
- 聊天响应新增 `reportTriggerLevel` 与 `reportActionText`，前端建议优先用这两个字段决定按钮文案与强调程度

## 2. 前端侧边栏状态机设计

### 2.1 四阶段定义

```typescript
type ReportStage = 'idle' | 'patient' | 'symptom' | 'review' | 'generated';
```

| 阶段 | 显示文案 | 进度 |
|---|---|---|
| `idle` | Waiting for patient... | 0% |
| `patient` | PATIENT INFO — Verified identity | 25% |
| `symptom` | SYMPTOM ANALYSIS — Data processing... | 50% |
| `review` | MEDICAL REVIEW — Reviewing findings... | 75% |
| `generated` | FINAL REPORT — Report Ready | 100% |

### 2.2 状态推进规则：only-forward

状态只允许前进，不允许后退：

```typescript
const STAGE_ORDER = { idle: 0, patient: 1, symptom: 2, review: 3, generated: 4 };

function advanceReportStage(newStage: ReportStage) {
  if (STAGE_ORDER[newStage] > STAGE_ORDER[currentStage]) {
    currentStage = newStage;
  }
}
```

### 2.3 后端信号 → 前端阶段映射

```
┌─────────────────────────────────────────────────────────────┐
│ 触发条件（后端信号）                   前端阶段推进           │
├─────────────────────────────────────────────────────────────┤
│ POST /users/profile 200 返回        → advanceReportStage   │
│ 拿到 userId + sessionId             ('patient')            │
│                                                             │
│ 第1轮 SSE done 事件到达             → advanceReportStage   │
│ (即 chatStream onDone 首次触发)     ('symptom')            │
│                                                             │
│ done.data.reportAvailable === true  → advanceReportStage   │
│ (后端返回 suggested/recommended/    ('review')             │
│ urgent 等报告触发信号)                                       │
│                                                             │
│ GET /reports/{sessionId} 返回       → advanceReportStage   │
│ ready === true && report !== null   ('generated')          │
└─────────────────────────────────────────────────────────────┘
```

### 2.4 详细数据流

#### Phase 1: idle → patient

```
用户填写 Name/Age/Gender → 点击 "Start Consultation"
  → POST /api/v1/users/profile
  → 200: { userId, sessionId, welcomeMessage }
  → store.setSession(userId, sessionId, welcomeMessage)
  → store.advanceReportStage('patient')     ← 侧边栏第1个 ✅
  → store.setOnboardingStatus('completed')
  → OnboardingOverlay 淡出
```

#### Phase 2: patient → symptom

```
用户发送第一条消息
  → POST /api/v1/chat/completions (Accept: text/event-stream)
  → SSE: event:meta → event:chunk* → event:done
  → onDone 回调:
      store.incrementChatRound()
      store.advanceReportStage('symptom')    ← 侧边栏第2个 ✅（或脉冲动画）
```

#### Phase 3: symptom → review

```
用户继续多轮问诊
  → 每轮 SSE done 到达时检查 data.reportAvailable
  → 当 reportAvailable === true:
      store.advanceReportStage('review')     ← 侧边栏第3个激活
      侧边栏显示 "Generate Report" 按钮
  → 同时可结合 data.reportTriggerLevel:
      urgent       → 高亮提示“建议立即生成报告协助就医”
      recommended  → 常规强调“建议生成报告”
      suggested    → 温和提示“可按需生成报告”
```

#### Phase 4: review → generated

```
用户点击侧边栏 "Generate Report" 按钮
  → GET /api/v1/reports/{sessionId}
  → 200: { ready: true, report: { title, riskLevel, summary, ... } }
  → store.setReport(report)
  → store.advanceReportStage('generated')   ← 侧边栏第4个 ✅
  → 显示报告摘要卡片 + Download 按钮
  → 侧边栏顶部显示绿色 "Generated" Badge
  → 点击 Download:
      GET /api/v1/reports/{sessionId}/pdf
      → 200 application/pdf
      → 浏览器下载 medical-report-{sessionId}.pdf
```

## 5. SSE 解析协议（不变）

前端使用 `fetch` + `ReadableStream` 解析。

状态机：`meta → chunk* → done/error`

```typescript
// services/api.ts
export async function chatStream(
  payload: ChatCompletionRequest,
  handlers: {
    onMeta?: (sessionId: string) => void;
    onChunk?: (delta: string, index: number) => void;
    onDone?: (data: ChatCompletionResponse) => void;
    onError?: (code: string, message: string) => void;
  }
): Promise<void>
```

前端打字机效果通过 16ms interval 缓冲引擎实现，不依赖后端推送速度。
