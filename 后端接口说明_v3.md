**不需要改的**
- URL 不变：
- `POST /v1/users/profile`
- `POST /v1/chat/completions`
- `GET /v1/reports/{sessionId}`
- `POST /v1/reports/{sessionId}/location`
- `GET /v1/reports/{sessionId}/pdf`

**推荐前端流程**
- 1. 调 `POST /v1/users/profile` 拿到 `userId + sessionId`。
- 2. 调 `POST /v1/chat/completions` 展示聊天回复。
- 3. 如果返回 `reportAvailable === true`，显示“可生成/可查看报告”入口。
- 4. 用户点击查看报告时，调 `GET /v1/reports/{sessionId}`。
- 5. 如果用户授权定位，先调 `POST /v1/reports/{sessionId}/location`，再重新调 `GET /v1/reports/{sessionId}` 刷新医院和路线。
- 6. 导出时直接调 `GET /v1/reports/{sessionId}/pdf`。

```ts
    | "CHAT_RUNTIME_ERROR"
    | "MODEL_UPSTREAM_UNAVAILABLE"
    | "INTERNAL_ERROR";

// done 事件 payload 就是 ChatCompletionResponse
export type ChatSseDoneEvent = ChatCompletionResponse;
```

**SSE 使用约束**
- `meta`：只含 `sessionId`
- `chunk`：增量文本
- `done`：完整 `ChatCompletionResponse`
- `error`：错误对象
- 前端不要再假设 SSE 结束前会附带报告预览
